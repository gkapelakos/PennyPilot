PennyPilot Release Notes
Version: alpha.1.13
Release Date: 2026-01-14

Overview
This update introduces significant functional enhancements and a complete redesign of the initial user experience to prepare the platform for beta release. Key focus areas include advanced budgeting logic, financial intelligence, and improved data portability.

Core Interface Redesign
- Redefined the startup experience with a unified landing interface.
- Implemented global currency selection during the initialization phase.
- Redesigned the onboarding sequence with improved visual fidelity and smoother transitions.
- Integrated automated language and currency persistence.

Budgeting and Financial Logic
- Implemented budget rollover functionality allowing unspent funds to carry forward to the next period.
- Added support for multiple budget cycles: Weekly, Bi-weekly, Monthly, and Yearly.
- Enhanced Safe-to-Spend calculations to factor in carry-over amounts and upcoming obligations.
- Improved subscription lifecycle tracking for enhanced accuracy.

Advanced Financial Intelligence
- Introduced the Insights Service for complex financial analytics.
- Implemented automated anomaly detection to identify significant changes in category spending.
- Added month-over-month fiscal comparisons and top-category distribution analysis.
- Developed "Merchant Normalization Learning" which automatically creates standardization rules based on manual user edits.

Security and Privacy
- Introduced Sensitive Data Masking, allowing users to obscure financial figures in public environments.
- Enhanced biometric authentication integration for increased reliability.
- Optimized local-first data isolation protocols.

Data Management and Portability
- Implemented a new Data Export Service.
- Added functionality to export transaction history to standardized CSV format.
- Integrated secure system sharing for exported fiscal documentation.

Technical Maintenance
- Version incremented to alpha.1.13.
- Removed deprecated language selection components.
- Optimized Riverpod provider tree for new service integrations.
- Refined database schema for Budget and Transaction models.
